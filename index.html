HTML

  • The head element contains information about the webpage
  • The body element represents the visible content to the user
  • The ul element is used for unordered lists
  • The li element is used for lists
  • The img element contains information about images that are displayed on the webpage.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The p element represents a paragraph or block of text.
  • Further information on HTML Tags is available here

CSS

  • Inline CSS apply to a specific HTML tag. They are useful for permanent changes but are less flexible than external and internal style sheets, as each inline style must be edited individually.
  • Internal CSS contains CSS rules for the webpage in the head element of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code.
  • External CSS allows developers to keep all their CSS rules in a separate file which makes design changes easier. The style sheet is connected to the HTML by adding the <link> element in the HTML's <head>.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content. This is sometimes referred to as the CSS box model.
  • Further information on the box model is available here
  • Using class selecter .card allows you to style only the elements you wish to.

Git

  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch to the staging area. -A indicates that we want to add all changes
  • git commit -m "add commit description": commit command and commit message
  • git pull: pull a branches modifications into the local environment
  • git pull: pushes changes to the remote branch
